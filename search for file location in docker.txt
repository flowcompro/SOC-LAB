Here are some steps for changing the admin password in Wazuh:
Run the script with the `-u` option and indicate the new password with the `-p` option
The password must have a length between 8 and 64 characters and contain at least one upper case letter, one lower case letter, a number and one of the following symbols
For the user `wazuh-wui`, modify the value of `API_PASSWORD` in `docker-compose.yml`
Modify the password value in the `single-node/config/wazuh-dashboard/wazuh.yml` file with the password you set in `API_PASSWORD` of `docker-compose.yml` file
Start the stack: `docker-compose up -d`
Remember to update the new password in `/etc/filebeat/filebeat.yml` and `/etc/kibana/kibana.yml` if necessary
The admin and kibanaserver users are set to read-only as security measures, so it is not possible to change the password using the WUI and it must be changed in the configuration files.
